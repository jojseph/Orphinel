LEXOR

A strongly-typed programming language built to teach Senior High School students the fundamentals of programming.

LEXOR is implemented as a pure interpreter — code runs directly without a separate compilation step, making it well-suited for learning, experimentation, and quick feedback.

The language emphasises readability through capitalised reserved words and explicit block markers like START SCRIPT and END SCRIPT, helping new programmers visualise program structure clearly.

At a glance

  • Strongly typed — every variable has a declared type
  • Pure interpreter — no compile step
  • Capitalised reserved words for visual clarity
  • Explicit block delimiters for every structure

Your First Program

Every LEXOR program follows the same shape. Here is the smallest meaningful program:

lexor 
SCRIPT AREA
START SCRIPT
    PRINT: "Hello, LEXOR"
END SCRIPT
Hello, LEXOR

Three things are happening here:

  1. SCRIPT AREA declares this file as an executable LEXOR script.
  2. START SCRIPT and END SCRIPT mark the boundaries of your code.
  3. PRINT: writes text to the output device.

Program Structure

A LEXOR program has three required parts, in this order:

lexor 
SCRIPT AREA              %% 1. Script declaration
START SCRIPT             %% 2. Begin script block
    DECLARE INT x = 10   %% 3a. Variable declarations come FIRST
    DECLARE CHAR ch = 'A'
    x = x + 5            %% 3b. Then executable code
    PRINT: x
END SCRIPT

Rules

  • Every program starts with SCRIPT AREA.
  • All code lives between START SCRIPT and END SCRIPT.
  • Variable declarations must come first — they cannot appear after executable code.
  • Each line contains exactly one statement.
  • All reserved words are CAPITALISED and cannot be used as variable names.
Why declarations first? Because LEXOR is strongly typed, the interpreter needs to know the type of every variable before it sees any expression that uses it.

Comments

Comments begin with %% and continue to the end of the line. They can be placed anywhere in your program.

lexor 
%% This entire line is a comment
SCRIPT AREA
START SCRIPT
    DECLARE INT x = 10  %% Comments work after statements too
    PRINT: x
END SCRIPT

Variable Names

LEXOR variable names follow these rules:

  • Case sensitive — count and Count are different variables.
  • Must start with a letter (a–z, A–Z) or an underscore (_).
  • Followed by any combination of letters, underscores, or digits.
  • Cannot use reserved words (PRINT, IF, INT, etc.) as variable names.

Data Types

LEXOR has four primitive data types. Every variable must be declared with an explicit type.

TypeDescriptionSizeExample
INTWhole number, no decimals4 bytesDECLARE INT x = 10
FLOATNumber with a decimal part4 bytesDECLARE FLOAT pi = 3.14
CHARA single character—DECLARE CHAR ch = 'A'
BOOLLogical TRUE or FALSE—DECLARE BOOL ok = "TRUE"

Declaring multiple variables

You can declare several variables of the same type on a single line:

lexor 
DECLARE INT x, y, z = 5

This declares three INT variables. x and y are uninitialised; z starts at 5.

Logical operators take BOOL expressions and return a BOOL result.

DECLARE BOOL result = "FALSE"

Output: PRINT

PRINT: writes formatted output to the screen.

lexor 
PRINT: "Hello, World"
PRINT: x
PRINT: x & " items: " & total

Multiple values are joined together using the concatenator &. See the Special Characters section for the full set of formatting tools.

Input: SCAN

SCAN: reads values from the user into one or more variables.

Syntax

syntax 
SCAN: <variableName>[, <variableName>]*

Example

lexor 
DECLARE INT x, y
SCAN: x, y
PRINT: x + y

When this runs, the user types two values separated by a comma — e.g. 10, 20 — and the program prints 30.

Special Characters

LEXOR provides three special print constructs that work alongside PRINT:.

SymbolMeaning
&Concatenator — joins values together into one output
$Newline (carriage return) — moves to the next line
[ ]Escape code — outputs the literal character inside the brackets
lexor 
DECLARE INT x = 5
DECLARE CHAR a = 'c'
PRINT: x & "TRUE5" & $ & a & [#] & "last"
5TRUE5 c#last

If / Else

The condition inside (...) must be a BOOL expression. Each branch is wrapped in START IF and END IF.

Simple if

lexor 
IF (x > 0)
START IF
    PRINT: "positive"
END IF

If / else

lexor 
IF (x > 0)
START IF
    PRINT: "positive"
END IF
ELSE
START IF
    PRINT: "non-positive"
END IF

If / else if / else

lexor 
IF (x > 0)
START IF
    PRINT: "positive"
END IF
ELSE IF (x < 0)
START IF
    PRINT: "negative"
END IF
ELSE
START IF
    PRINT: "zero"
END IF

For Loop

The three parts in parentheses are initialisation, condition, and update, separated by commas.

lexor 
FOR (i = 1, i <= 10, i = i + 1)
START FOR
    PRINT: i & $
END FOR

This loop prints the numbers from 1 to 10, each on its own line.

Repeat When

REPEAT WHEN runs as long as the condition stays TRUE — equivalent to a while loop in other languages.

lexor 
DECLARE INT n = 1
REPEAT WHEN (n <= 5)
START REPEAT
    PRINT: n & $
    n = n + 1
END REPEAT
1 2 3 4 5

Sample Programs

Three complete examples drawn from the LEXOR language specification.

1. Mixed variables and output

lexor 
%% this is a sample program in LEXOR
SCRIPT AREA
START SCRIPT
    DECLARE INT x, y, z = 5
    DECLARE CHAR a_1 = 'n'
    DECLARE BOOL t = "TRUE"
    x = y = 4
    a_1 = 'c'
    %% this is a comment
    PRINT: x & t & z & $ & a_1 & [#] & "last"
END SCRIPT
4TRUE5 c#last

2. Arithmetic operations

lexor 
SCRIPT AREA
START SCRIPT
    DECLARE INT xyz, abc = 100
    xyz = ((abc * 5) / 10 + 10) * -1
    PRINT: [[] & xyz & []]
END SCRIPT
[-60]

The [[] and []] constructs print literal [ and ] characters around the value of xyz.

3. Logical operations

lexor 
SCRIPT AREA
START SCRIPT
    DECLARE INT a = 100, b = 200, c = 300
    DECLARE BOOL d = "FALSE"
    d = (a < b AND c <> 200)
    PRINT: d
END SCRIPT
TRUE
